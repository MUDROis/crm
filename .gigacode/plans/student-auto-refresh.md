# План: Автообновление списка учеников при создании

## Проблема
При создании нового ученика страница не обновляется автоматически. Пользователю приходится вручную перезагружать страницу, чтобы увидеть нового ученика в списке.

## Анализ текущей реализации

### Текущий поток:
1. **StudentForm.tsx** - после сохранения вызывает `onSaved()` callback
2. **page.tsx** - `handleSaved()` увеличивает `refreshKey`, что должно вызвать пересоздание StudentList
3. **StudentList.tsx** - использует `useSupabaseQuery` с кэшированием
4. **useSupabaseQuery.ts** - имеет механизм `refetch()` для принудительного обновления

### Проблема:
- `StudentList` получает `refreshKey` как prop, но не использует его для вызова `refetch()`
- Кэш в `useSupabaseQuery` не инвалидируется при изменении ключа
- Механизм автообновления не срабатывает корректно

## Решение

### Изменяемые файлы:
1. **StudentList.tsx** - добавить эффект для вызова `refetch()` при изменении `refreshKey`
2. **page.tsx** - передать `refreshKey` в `StudentList` и изменить логику `handleSaved`
3. **useSupabaseQuery.ts** - улучшить механизм инвалидации кэша при ручном вызове `refetch()`

## Шаги реализации

### Шаг 1: Изменить StudentList.tsx
- Принять `refreshKey` как prop
- Использовать `useEffect` для вызова `refetch()` при изменении `refreshKey`
- Убедиться, что `refetch()` корректно инвалидирует кэш

### Шаг 2: Изменить page.tsx
- Передавать `refreshKey` в `StudentList` через prop
- Убедиться, что `handleSaved()` правильно обновляет ключ

### Шаг 3: Проверить useSupabaseQuery.ts
- Убедиться, что `refetch()` верно очищает кэш
- Проверить, что при `refreshCounter > 0` данные перезагружаются

## Ожидаемый результат
После создания нового ученика список автоматически обновится без перезагрузки страницы.
